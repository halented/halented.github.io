Making a Site...Pretty!

posted march 8th, 2019
by hal


From Bootstrap’s own mouth, it will allow you to:

“Build responsive, mobile-first projects on the web with the world’s most popular front-end component library.”

A lot of that sentence can just wash over you, but let’s break it down a bit:

“responsive, mobile-first” means that bootstrap is intended to create websites which are reactive to the type of device with which they are being viewed. If your screen is tiny, the site will still be able to present itself in a manageable way.

“world’s most popular” seems like an opinion but we’ll let that fly, and —

“front-end component library” is the most important part.

Bootstrap is a huge, user-friendly library of built-out front-end methods that you can use, or not use, at your discretion to create the look and feel that you need for your site.

HTML & CSS’s whole main gig is stylizing things through classes. Bootstrap gives you tons and tons of classes which are already stylized and ready to roll. All you have to do is give your project access to Bootstrap’s tools, and use ‘em, babey.

Because this website is hosted on GitHub Pages, I’m using HTML primarily, so I was able to give my project access to Bootstrap using the template provided on their site. The simplest way of using the template is to copy it and pop it into your index.html file (wherever that may be in your project). More detailed instructions on using Bootstrap in this way can be found on their site here, but we are Rubyists, so we'll take a closer look at how to use it with Rails.

The best way to incorporate Bootstrap into your rails project is with, you guessed it! Gems.

Once you are inside of your rails project, visit your friendly Gemfile and enter:

gem 'bootstrap', '~> 4.3.1'

This is the latest/greatest version of bootstrap. You could just use bootstrap without specifying, but the way that it interacts with Rails has changed over time and will surely change in the future, so version control is best practice. This 4.3.1 specification is the appropriate version to use with Rails 5 and above.

Bootstrap also uses Javascript. It doesn't necessarily require you to understand anything about Javascipt, but you will also need to require the appropriate gem using:

gem 'jquery-rails'

With that completed, you will also need to change let your application know that Bootstrap intends to use Sass rather than regular ol' CSS, so head over to app/assets/stylesheets and change the exension for your application.css file to application.scss. It's still CSS, but just, sassier.

Once you've changed the file extension to understand Bootstrap code, you will then need to import some code into the file itself. It's super tricky. You open the 'application.scss' file and type the following at the bottom:

@import "bootstrap";

Code = imported! The final step is related to the aforementioned Javascript use, so we're just going to bust out our copy/paste skills to pop the following code into app/assets/javascripts/application.js:

//= require jquery3 //= require popper //= require bootstrap-sprockets

Now all of this is stuff that's detailed in informational page for Bootstrap's Ruby gem which you can find on GitHub here. With that detailed, we can go into Bootstrap's abilities a little bit.

As I stated in the opening, the point of Bootstrap is to provide a bunch of pre-made style options so that you can spend as little time as possible on your front-end and more time on building your cool, original website with unique functionality. It's similar to Rails in that its functionality takes care of so very much, that you need to set some time apart to become familiar with it and really learn Bootstrap before you can dive into utilizing it as the full, powerful tool library that it is.

I've included on this website a nav-bar link which will show you a slew of basic examples of Bootstrap's influence on the colors and fonts for your page. If you click on it, the first thing you will notice is the annoyingly bright background. One of the reasons that Bootstrap is so popular is because it is designed to be used with "reactive breakpoints" -- the color of the background you are seeing is actually determined by the size of your screen in pixels, and that is customizable. If you take a peek into the code for this, you will see:

<style> body{ background: aqua; } @media screen and (min-width:576px){ body{ background: red; } } @media screen and (min-width:768px){ body{ background:blue; } } @media screen and (min-width:992px){ body{ background:green; } } @media screen and (min-width:1200px){ body{ background:yellow; } } </style>

The @media variable you see it just Bootstrap talking to itself, by providing a keyword to indicate that the size of the screen should be evaluated. Allowing you to command the background's reaction at specific sizes gives you a simple, powerful way to wow your users.

The other vital component to Bootstrap is its use of grids. The grid system keeps your elements organized and prevents you from overlapping your own code. If we take a look back at the example page, you can see the varying changes in position, margins, padding, and other elements.

The grid system is build out of rows and columns, as you might imagine. You should begin your grid with a div tag marked as a "container" or "container-fluid" class, which are two of the classes granted to you by Bootstrap. The only difference between a container and a fluid container is that the fluid one will hit the edges of the window (or "flood" it), whereas the regular container is more boxy.

<div class="container"> <div class="row"> <div class="col"> <!--some code!--> </div> </div> </div>

The grid system is built out of 12 parts. As you can see here I've included a few more divisions for a row and a column. Within these divisions you can specify how you want that row or column to act. The grid will not allow you to include more than 12 colums per row, and it will default to fill all twelve columns if you tell it there is only one column in that row. Here is an example of some code where we have specified the sizes of the columns:

<div class="container"> <div class="row"> <div class="col-9 bg-primary">I am 9 of the twelve</div> <div class="col-3 bg-info">I am 3 of the twelve</div> </div> </div>

I gave these classes a few more specifications so that we can see the columns properly -- the "bg-primary" part will just tell it to give that division a background color. Check out the evaluated code below.

I am 9 of the twelve
I am 3 of the twelve

I have hosted the code behind the example page on a JSFiddle which can be found here. This gives you a basic idea of what the class tags are doing and how you might begin to manipulate them with the grid. That said, Bootstrap is an expansive, fun-as-hell system that would take quite a while to grasp in its entirety, so I encourage anyone reading this to fiddle away! There are tons of tips and walk throughs available from Bootstrap's own documentation here, and you can also find a wealth of tutorial videos which go in much greater detail on various websites. As a personal recommendation, there is an extremely detailed basics video you can find from the Coding Addict Youtube channel right here as well. Happy coding!
